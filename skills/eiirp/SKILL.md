# EIIRP

Use this skill after a substantial work session when outputs, notes, docs, and brain artifacts need to be organized.

## Workflow

1. Inventory outputs:
   - code changes
   - docs
   - reports
   - brain-worthy notes
   - loose artifacts
2. Classify each item:
   - commit/package artifact
   - report
   - source
   - concept
   - idea
   - task
3. Use `brain-taxonomist` for brain writes.
4. Use `schema-author` if no existing category fits.
5. Run validation appropriate to the moved or created artifacts.

## Guardrails

- Do not move user files silently.
- Do not rewrite untracked `brain/` corpus files as cleanup unless asked.
- Keep organization decisions reversible.
